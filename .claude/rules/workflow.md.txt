# Workflow Rules

## Git
- Branch từ `develop`, không commit thẳng vào `main`
- Commit message format: `<type>: <description>` (feat/fix/chore/refactor/docs)
- Co-author với Claude nếu Claude viết code

## Khi thêm feature mới
1. Tạo entity trong `domain/<feature>/`
2. Tạo Repository interface trong `domain/<feature>/`
3. Tạo DTOs + Exceptions trong `app/<feature>/`
4. Tạo `CommandHandler` và/hoặc `QueryHandler` trong `app/<feature>/handler/`
5. Tạo Controller trong `infrastructure/web/<feature>/`

## Khi viết Controller
- Luôn dùng explicit name: `@PathVariable("id")`, `@RequestParam("status")`
- Không inject Handler trực tiếp vào Controller nếu có thể dùng interface
